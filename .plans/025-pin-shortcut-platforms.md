# pin / unpin ショートカットの OS 別設定計画

## 目的

- pin / unpin のショートカットキーを OS ごとに変更する。
- macOS では `ctrl+command+p`、Windows では `ctrl+alt+p` を使用する。
- 初回起動時の状態を unpin にする。
- pin 状態の場合は side toc が表示されているサイドバーの右下に `📌` を表示する。
- 起動時に保存済み状態があっても pin 状態で開始しないようにする。
- pin / unpin 時の `Pinned` / `Unpinned` 通知メッセージを表示しない。

## 修正方針

1. [x] 既存の pin / unpin ショートカット定義と処理箇所を確認する。
2. [x] macOS と Windows で異なるデフォルトキー割り当てになるように設定を変更する。
3. [x] 初回起動時の pin 状態が保存値やデフォルト設定から pin になっていないか確認し、必要に応じて unpin を初期値にする。
4. [x] pin / unpin の通知を UI 状態に反映し、pin 状態のときだけサイドバー右下にピン表示を追加する。
5. [x] 関連するテストまたは型チェックを実行し、影響範囲を確認する。
6. [x] 保存済み window state の `always_on_top` を起動時に復元しないように修正する。
7. [x] `h1::before` のピン表示を削除し、サイドバー右下表示へ変更する。
8. [x] pin / unpin の通知メッセージを削除し、状態更新のみ行う。

## 確認事項

- Linux など未指定の環境は、既存の挙動を維持するか、既存のデフォルト設定に従う。
- ユーザー設定済みの keymap がある場合は、既存の上書きルールを尊重する。
