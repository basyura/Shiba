# アプリケーション概要

## 目的

Shiba は Markdown ファイルを WebView 上でプレビューするデスクトップアプリケーションである。Rust 側が CLI、設定、Markdown 解析、ファイル監視、永続化、WebView 連携を担当し、TypeScript/React 側がプレビュー表示、検索 UI、履歴 UI、アウトライン、キーボードショートカットを担当する。

## 起動処理

1. `main` は CLI 引数を `Options::parse` で解析する。
2. `--help` の場合は使用方法、`--version` の場合はパッケージバージョンを標準出力へ出す。
3. 通常起動では `--debug` に応じてログレベルを設定し、`run(options)` を呼ぶ。
4. `run` は `--no-watch` の有無に応じて、実ファイル監視または no-op 監視を選択して `Shiba` を起動する。
5. Windows のリリースビルドでは `windows_subsystem = "windows"` を指定し、コンソールウィンドウを出さない。

## CLI オプション

| オプション | 仕様 |
| --- | --- |
| `-t`, `--theme THEME` | ウィンドウテーマを `system`、`dark`、`light` から選ぶ。設定ファイルより優先される。 |
| `--no-watch` | ファイル変更監視を無効化する。 |
| `--no-restore` | 前回のウィンドウ状態復元を無効化する。 |
| `--generate-config-file` | 既定設定ファイルを生成する。既存ファイルは上書きされる。 |
| `--config-dir PATH` | 設定ディレクトリを指定する。 |
| `--data-dir PATH` | 永続データディレクトリを指定する。 |
| `--debug` | デバッグログとデバッグ機能を有効化する。 |
| `-h`, `--help` | ヘルプを表示する。 |
| `--version` | バージョンを表示する。 |

## 起動時パス引数

- 存在する最初のファイルパスは初期プレビュー対象になる。
- 2 個目以降のパス、存在しないパス、ディレクトリは監視対象になる。
- 存在するパスは canonicalize される。
- 存在しないパスはカレントディレクトリ基準の絶対パスとして扱われる。

## 初期表示

- Renderer から `init` メッセージを受けると、Rust 側は設定、スクロール設定、検索設定、最近使ったファイル、ホームディレクトリ、ウィンドウ外観を UI 側へ送る。
- 初期ファイルまたは保留中の OpenedFiles がある場合は Markdown として開く。
- 開く対象がない場合は Welcome 画面を表示する。
- ウィンドウは初期コンテンツが準備された後に表示される。暗色テーマで白いウィンドウが一瞬出ることを避けるためである。

## ファイルを開く条件

- 開く対象は通常ファイルである必要がある。
- 拡張子は設定 `watch.file_extensions` に一致する必要がある。
- 相対パスは canonicalize できる場合だけ開かれる。
- Markdown 以外のローカルリンクは OS の既定アプリケーションで開かれる。

## 履歴

- プレビューに成功したファイルは履歴へ追加される。
- 現在のファイルと同じパスは重複追加されない。
- 履歴の最大件数は `preview.recent_files` で制御される。
- 新しいファイルを開いた時点で、現在位置より先の forward 履歴は破棄される。
- 終了時および履歴追加時に永続データへ保存される。

## 永続データ

- 既定のデータディレクトリは OS のアプリケーションデータディレクトリ配下の `Shiba` である。
- `--data-dir` が指定された場合はそのディレクトリを使う。ただし存在するディレクトリでない場合は永続化されない。
- 最近使ったファイルは `recent_files.json` に保存される。
- ウィンドウ状態は `window.restore` が有効な場合に `window.json` に保存される。
