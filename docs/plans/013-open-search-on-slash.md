# / キーで検索ウインドウを表示する計画

## 目的

- `/` 入力時に検索ウインドウを表示する。
- macOS の `cmd+f`、Windows/Linux の `ctrl+f` と同じ検索開始挙動に揃える。

## 調査方針

- 既存の検索開始処理とキーマップ処理を確認する。
- `cmd+f` / `ctrl+f` がどの経路で検索ウインドウを開くか確認する。
- `/` が入力欄などで通常入力として使われる場面を邪魔しない条件を確認する。

## 修正方針

- 既存の検索開始アクションを再利用し、`/` キー押下時に同じ処理を呼び出す。
- 入力要素にフォーカスがある場合は `/` を通常入力として扱う。
- 既存の検索中の次候補・前候補移動の挙動には影響を与えない。

## 検証

- `cargo test config` で設定読み込みと既定キーマップ更新を確認する。
- 必要に応じて、検索ウインドウ表示の手動確認手順を整理する。

## 実施内容

- 既定キーマップに `/: Search` を追加する。
- 配布用の `src/assets/default_config.yml` にも同じ設定を追加する。
- `/` 追加前の既定キーマップをアップグレード対象として残し、既定設定を使っている既存ユーザーにも反映されるようにする。
