# Open File ダイアログの初期フォルダ調整

## 背景

現在の `Open File` は、常に `config.yml` の `dialog.default_dir` を基準にファイル選択ダイアログを開いている。

既にファイルを開いている場合は、次に別ファイルを開く操作では現在のファイルと同じ階層を開き、同階層のファイルをカーソルキーで選択できる状態にする方が自然。

`dialog.default_dir` が `null` の場合、現在は `env::current_dir()` が使われる。
Windows ではその値がそのまま使われ、`Documents` などへの追加フォールバックはない。
macOS ではカレントディレクトリが `/` の場合のみ `~/Documents` にフォールバックする。

現状では、ファイルを開いていてもそのファイルの親ディレクトリは参照されない。

## 修正方針

- `Open File` の初期ディレクトリ決定処理を追加する。
- 現在表示中の履歴ファイルがある場合は、その親ディレクトリを初期ディレクトリとして渡す。
- macOS では初期ファイル名を渡さず、親ディレクトリだけを初期ディレクトリとして渡す。
- macOS で現在ファイル自身のパスや初期ファイル名を渡すと、親ディレクトリにフォーカスが残り、カーソルキーで同階層ファイルへ移動できないため避ける。
- ただし Welcome 表示などで実際にプレビュー中のファイルがない場合は、履歴があっても現在ファイルとして扱わない。
- 現在表示中のファイルがない場合、または親ディレクトリが取得できない場合は、従来通り `config.yml` の `dialog.default_dir` を使う。
- `Open Directory` は今回の要件外なので、従来通り `dialog.default_dir` を使う。

## 修正後の優先順位

1. 実際にファイルをプレビュー中の場合は、その親ディレクトリを開く。
2. ファイルを開いていない場合は、`dialog.default_dir` を使う。
3. `dialog.default_dir` が `null` の場合は、従来通り `env::current_dir()` を使う。
4. macOS で `env::current_dir()` が `/` の場合のみ、従来通り `~/Documents` にフォールバックする。

Windows では `dialog.default_dir` が `null` の場合、従来通り `env::current_dir()` が最終的なフォールバック先になる。

## 変更予定ファイル

- `src/shiba.rs`
  - `open_file` で使う初期パスを、現在ファイルの親ディレクトリ優先に変更する。
  - 必要に応じて小さな補助メソッドを追加する。
  - 初期ディレクトリ決定処理をテストしやすい小さな関数に切り出す。
  - 現在ファイルがある場合とない場合の単体テストを追加する。
  - 履歴があってもプレビュー中でなければ `dialog.default_dir` にフォールバックする単体テストを追加する。
- `AGENTS.md`
  - 修正と合わせて対応するテストコードを追加・更新する方針を `Testing Guidelines` に追記する。
- `src/dialog.rs`
  - ファイルダイアログへ初期ディレクトリを渡す。
