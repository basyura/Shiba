# 右クリックメニューに Editor を追加する計画

## 目的

- 右クリックメニューに `Editor` メニューを追加する。
- 現在開いている Markdown ファイルを、設定された任意のエディタで開けるようにする。
- エディタの指定は `config.yml` の `editor.path` で変更できるようにする。

## 調査方針

- 既存の右クリックメニュー実装を確認する。
- Markdown ファイルの現在パスを保持・参照している箇所を確認する。
- 設定ファイルの読み込み構造と `default_config.yml` の項目追加方法を確認する。
- OS ごとの外部コマンド実行方法を確認し、既存の open 系処理に合わせる。

## 修正案

- 設定に `editor.path` を追加し、以下の形式で指定できるようにする。

```yaml
editor:
  path: path_to_editor
```

- デフォルト設定には未指定または既定エディタ相当の値を追加し、既存利用者の挙動を壊さない。
- 右クリックメニューに `Editor` を追加し、選択時に現在の Markdown ファイルパスを Rust 側へ渡す。
- Rust 側で設定されたエディタコマンドとファイルパスを使って外部プロセスを起動する。
- エディタ未設定、ファイル未オープン、起動失敗時の扱いを既存の通知・ログ方針に合わせる。

## 確認項目

- `config.yml` に指定したエディタで現在の Markdown ファイルを開ける。
- エディタ未指定時に既存のファイル表示や右クリックメニューが壊れない。
- TypeScript と Rust の型・フォーマットが通る。
- 変更範囲に応じて `cargo test` または focused test を実行する。

## 実施結果

- 右クリックで既存のネイティブメニューを開く UI 処理を追加した。
- メニューに `Editor` 項目を追加した。
- `config.yml` の `editor.path` で外部エディタの実行ファイルを指定できるようにした。
- `Editor` 選択時に現在開いている Markdown ファイルをエディタの引数として渡して起動するようにした。

## 追加修正方針

- `open_menu` はメニューボタンとショートカット用のアプリメニュー表示として維持する。
- 右クリック用に `open_context_menu` を追加し、アプリメニュー全体とは別経路にする。
- 右クリック専用メニューは従来の `Reload` と `Inspector` に `Editor` を加えた最小構成にする。
- 右クリック専用メニューの `Inspector` は既存の DevTools 表示処理に接続する。

## `.app` 指定への追加対応

- macOS で `editor.path` が `.app` バンドルを指す場合は、直接実行せず `open -a <app> <markdown>` で起動する。
- `.app` 以外のパスは従来どおり実行ファイルとして扱い、`<editor> <markdown>` で起動する。
- 起動コマンドの分岐をテストできる helper に分離する。

## MacVim の既存ウインドウ対応

- macOS で `editor.path` が `MacVim.app` を指す場合は、`open -a` ではなく MacVim の `mvim` コマンドを使う。
- `mvim --remote-silent <markdown>` で、起動済み MacVim があれば既存ウインドウに開く。
- MacVim.app 内の `mvim` が見つからない場合は従来の `.app` 起動にフォールバックする。
