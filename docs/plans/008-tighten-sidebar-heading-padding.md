# サイドバー見出しの左パディングを詰める

## 1. 課題

サイドバーでレベル1見出しを非表示にしたため、表示上の最上位であるレベル2見出しの左パディングが広く見える。

## 2. 対象箇所の確認

- `ui/components/SideBar.tsx` の `buttonStyle.padding` で左パディングに `${h.level}em` を指定している。
- レベル2見出しは `2em`、レベル3見出しは `3em` の左パディングになる。

## 3. 仕様整理

- レベル1見出しは引き続き非表示にする。
- 表示される最上位のレベル2見出しは `1em` の左パディングにする。
- レベル3以降は階層差が分かるよう、従来通り 1em ずつ増やす。

## 4. 実装方針

- 左パディングを `h.level` から `h.level - 1` に変更する。
- レベル1見出しはフィルタ済みのため、表示対象の左パディングは最低 `1em` になる。

具体的なコード案:

```ts
padding: `0 1em 0 ${h.level - 1}em`,
```

## 5. 影響確認

- レベル2見出しの左パディングが詰まることを確認する。
- レベル3以降の相対的なインデントが維持されることを確認する。

## 6. 動作確認方針

- TypeScript の型チェックを実行する。
- Prettier と ESLint を実行する。

## 7. 対応内容 (結果)

- `SideBar` の見出し左パディングを `h.level` から `h.level - 1` に変更した。
- レベル2見出しを `1em`、レベル3見出しを `2em` として、階層差を保ったまま全体を詰めた。
