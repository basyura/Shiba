# プレビュー選択テキストのコピー

## 背景

プレビュー本文を選択した状態で右クリックしても、現在のコンテキストメニューには選択文字列をコピーする項目がない。

## 修正案

1. React 側の右クリック処理で `window.getSelection()` から選択文字列を取得する。
2. 選択文字列が空でない場合は、IPC の `open_context_menu` に選択文字列を同梱する。
3. Rust 側の `OpenContextMenu` メッセージに選択文字列を受け取るフィールドを追加する。
4. コンテキストメニューに `コピー` 項目を追加する。
5. 右クリック時に選択文字列がある場合だけ `コピー` を有効化し、選択されたら受け取って保持している文字列をクリップボードへコピーする。
6. 選択文字列がない場合は `コピー` を無効化し、既存の Reload / Editor / Inspector の挙動は維持する。

## 変更予定ファイル

- `ui/components/App.tsx`
- `ui/ipc.ts`
- `src/renderer.rs`
- `src/wry/menu.rs`
- `src/shiba.rs`

## テスト方針

- `open_context_menu` メッセージが選択文字列ありでデシリアライズできることをユニットテストする。
- メニュー項目の追加により `コピー` が `MenuItem` に対応することを既存のメニュー処理で確認する。
- `cargo test --lib` と TypeScript 型チェックを実行する。
