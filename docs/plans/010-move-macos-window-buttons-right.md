# macOS 標準ウィンドウボタンを右上へ移動する計画

## 概要

- macOS の標準ボタンの見た目と動作は維持し、AppKit の `standardWindowButton` で取得したボタン位置だけを右上へ再配置する。
- Shiba は macOS で透明タイトルバーを使っているため、この方式で既存 UI のドラッグ領域を維持する。

## 変更内容

- `src/wry/webview.rs` に macOS 専用ヘルパーを追加し、閉じる/最小化/拡大ボタンを右端から一定マージンの位置へ移動する。
- ウィンドウ生成直後に再配置を実行する。
- ウィンドウリサイズ後も右上に追従するよう、リサイズイベントから再配置を呼び直す。
- Windows/Linux の挙動と UI 側の `WindowBar` は変更しない。

## テスト

- `cargo check --target aarch64-apple-darwin`
- 可能なら `npm run lint`
- `Shiba.app` 起動後、ボタン位置、クリック動作、リサイズ追従、ドラッグ/ダブルクリック最大化を目視確認する。

## 前提

- 独自描画ボタンにはせず、macOS の標準ボタンを維持する。
- macOS 標準とは異なる配置になるため、AppKit の将来変更で微調整が必要になる可能性がある。
